## 최종 결정: Baseline 유지

### 현재 방어 구조 (3층)

1. **F-Score 4점 필터** (스크리닝 단계) — 과거 대부분의 failure 배제
2. **`risk_guard.check_delisting_imminent`** — 매일 16:30 폐지 임박 30일 전 알림
3. **분기 리밸런싱 자동 교체** — 폐지 종목은 자연스럽게 다음 분기 선정에서 제거

### 수용된 리스크

| 지표 | 값 |
|------|-----|
| 노출률 | 32회 중 1회 (3.1%) |
| 단일 영향 | -5.00% |
| 연간화 | **-0.64%p CAGR** |

Baseline CAGR +18.76% 기준 -0.64%p → 실질 운용 CAGR ≈ **+18.1%**. 이 수준의 리스크를 제거하기 위해 +5%p 이상 희생할 가치는 없음.

## 학습 (Lessons Learned)

### 1. 단일 사례 회피를 위해 시스템 전체 성과를 희생하지 말 것

3.1% 노출률을 0%로 만들려고 CAGR을 18.76% → 6.58%로 낮추는 것은 명백한 손실. **기대값 기반 의사결정**이 필요.

### 2. 데이터 신선도가 알파의 핵심 원천

`strict_reporting_lag` 실험의 가장 큰 교훈: 최근 분기 데이터를 버리면 알파의 절반 이상 사라진다. 모멘텀·밸류 팩터 모두 **최근 실적 변화**를 감지해야 작동.

### 3. 확률적 사고

3.1% = 32분기 중 1건 = 8년간 1건. 이 한 건 때문에 8년간 더 낮은 수익률을 감수하는 것은 **리스크 회피 비용이 기댓값보다 크다**.

### 4. 이상 감지보다 분산 + 사후 대응

20종목 동일가중 분산으로 단일 종목 영향은 -5%에 국한됨. 사전 이상 감지보다 **사후 빠른 교체**(분기 리밸런싱)가 비용 효율적.

### 5. 방어 장치 도입의 5가지 조건 (POLICY.md)

- CAGR 손실 ≤ -1%p
- 폐지 회피 효과 명확
- Sharpe 하락 < 0.10
- Baseline 대비 종목 겹침률 ≥ 90%
- 8년 백테스트 + 구간별 안정성

# 모의투자 실전 테스트 계획

> 작성일: 2026-03-13
> 대상: 키움증권 모의투자 계좌 (`mockapi.kiwoom.com`)

## 1. 테스트 개요

| 항목 | 내용 |
|------|------|
| 목적 | 자동매매 시스템의 실전 환경 검증 |
| 환경 | 키움증권 모의투자 REST API |
| 기간 | 약 4~5주 (3단계) |
| 설정 | `IS_PAPER_TRADING=true` |

### 전체 일정

```
[1단계] 사전 검증 (1~2일)      ← 장 외 시간 가능
[2단계] 단발 실행 테스트 (1일)   ← 장 시간 필수 (09:00~15:30)
[3단계] 자동 운용 테스트 (1개월) ← 스케줄러 상주
```

---

## 2. 1단계: 사전 검증 (장 외 가능)

시간 제약 없이 진행할 수 있는 검증 항목입니다.

### 2.1 환경변수 확인

```bash
# .env 파일 필수 항목
IS_PAPER_TRADING=true
KIWOOM_APP_KEY=xxxxx
KIWOOM_APP_SECRET=xxxxx
KIWOOM_ACCOUNT_NO=xxxxxxxxxx
TELEGRAM_BOT_TOKEN=xxxxx
TELEGRAM_CHAT_ID=xxxxx
```

### 2.2 스크리닝 테스트

```bash
python scheduler/main.py --screen-only
```

체크리스트:
- [ ] N종목이 정상 선정되는지 (config.yaml의 `n_stocks` 값)
- [ ] 종목 리스트가 합리적인지 (금융주 제외, 유동성 충분 등)
- [ ] 에러 로그 없는지
- [ ] 텔레그램 알림 수신되는지

### 2.3 API 연결 테스트

```bash
python scheduler/main.py --dry-run
```

체크리스트:
- [ ] `mockapi.kiwoom.com` 연결 성공
- [ ] 토큰 발급 정상
- [ ] 계좌번호 인식
- [ ] 텔레그램 시작 메시지 수신

### 2.4 모의 계좌 잔고 확인

키움증권 HTS/MTS에서 직접 확인:
- [ ] 모의투자 계좌 자금 설정 완료
- [ ] 기존 보유 종목 없는 깨끗한 상태인지

---

## 3. 2단계: 단발 실행 테스트 (장 시간 필수)

**실행 시간**: 평일 09:30~14:30 권장 (장 시작/마감 직전은 피하기)

### 3.1 즉시 리밸런싱 실행

```bash
python scheduler/main.py --now
```

이 명령은 월말 체크를 스킵하고 즉시 전체 플로우를 실행합니다:

```
스크리닝(N종목 선정)
  → 현재 잔고 조회 (첫 실행: 보유 종목 없음)
  → 매도 (없음)
  → 매수 (N종목, 시장가 주문)
  → 텔레그램 결과 알림
```

### 3.2 체결 확인 체크리스트

| 확인 항목 | 방법 | 기대 결과 |
|----------|------|----------|
| 주문 접수 | 로그 확인 | N건 매수 주문 접수 |
| 체결 여부 | 키움 HTS/MTS | 전종목 전량 체결 |
| 체결 금액 | HTS 잔고 | 종목당 균등 배분 |
| 예수금 잔액 | HTS 잔고 | 투자금의 약 1% 이하 |
| 텔레그램 | 채팅 확인 | 매수 결과 리포트 수신 |
| DB 저장 | `data/quant.db` | Trade 테이블에 N건 기록 |

### 3.3 일간 리포트 확인 (15:35 자동 발송)

텔레그램으로 수신되는 내용:
- 총 평가금액, 투자 원금, 총 손익(%)
- 보유 종목 수익률 (수익률 높은 순)
- MDD, 상위 3종목 집중도

### 3.4 문제 발생 시 대처

| 문제 | 원인 | 조치 |
|------|------|------|
| 토큰 발급 실패 | APP_KEY/SECRET 불일치 | .env 재확인 |
| 주문 거부 | 장 마감 후 실행 | 다음 영업일 09:30 이후 재실행 |
| 일부 종목 미체결 | 유동성 부족 | 로그 확인 후 다음 달 대기 |
| 예수금 부족 | 모의 계좌 자금 부족 | 키움에서 모의투자 재신청 |
| 텔레그램 미수신 | 봇 토큰/채팅 ID 오류 | .env 재확인 |

---

## 4. 3단계: 자동 운용 테스트 (1개월)

2단계 성공 후 스케줄러를 상주 모드로 전환합니다.

### 4.1 스케줄러 시작

```bash
# 터미널에서 직접 (로그 확인 용이)
python scheduler/main.py

# 또는 백그라운드 실행 (Windows)
start /B python scheduler/main.py
```

스케줄러 동작:
- **매일 08:50**: 월말 영업일인지 확인 → 맞으면 리밸런싱
- **매일 15:35**: 일간 리포트 텔레그램 발송

### 4.2 일별 모니터링 체크리스트

매일 텔레그램 리포트 수신 시 확인:

| 항목 | 확인 내용 |
|------|----------|
| 총 평가금액 | 전일 대비 급변 없는지 |
| 당일 수익률 | 코스피 대비 과도한 괴리 없는지 |
| MDD | 서킷브레이커 접근 여부 |
| 종목 집중도 | 상위 3종목 > 15%면 주의 |
| 트레일링 스톱 | 개별 종목 -20% 이탈 발생 여부 |

### 4.3 월말 리밸런싱 관찰 (핵심)

한 달 운용 중 월말 리밸런싱이 자동으로 실행되는 것을 관찰합니다.

```
08:50 → 월말 영업일 확인 → 리밸런싱 시작
  → 스크리닝 (새 N종목 선정)
  → 기존 종목 중 탈락 종목 매도
  → 신규 종목 매수
  → 교체율 제한 (max_turnover_pct) 적용
  → 텔레그램 결과 알림
```

체크리스트:
- [ ] 월말에 자동으로 리밸런싱 실행되었는지
- [ ] 매도/매수 주문이 전량 체결되었는지
- [ ] 교체율 제한이 작동하는지
- [ ] 텔레그램 리포트에 교체 내역 표시

### 4.4 1개월 후 평가

| 평가 항목 | 기준 | 합격 |
|----------|------|------|
| 시스템 안정성 | 1개월간 에러 없이 운영 | 에러 0건 |
| 스케줄러 | 매일 리포트 발송 | 영업일 100% |
| 리밸런싱 | 월말 자동 실행 | 정상 체결 |
| 수익률 추적 | 코스피 대비 비교 가능 | 데이터 축적 |
| 텔레그램 | 모든 알림 정상 수신 | 누락 0건 |

## 5. 실전 전환 판단 기준

1개월 모의투자 테스트 완료 후, 다음 조건을 모두 만족하면 실전 전환을 검토합니다:

| 조건 | 기준 |
|------|------|
| 시스템 무장애 | 1개월간 오류 없음 |
| 체결률 | 주문 대비 체결 95% 이상 |
| 슬리피지 | 시장가 대비 0.3% 이내 |
| 리밸런싱 | 1회 이상 정상 수행 |
| 수익률 트래킹 | DB에 전 기간 데이터 축적 |

### 실전 전환 시 변경사항

```env
# .env 변경
IS_PAPER_TRADING=false         # true → false
KIWOOM_APP_KEY=실전키           # 실전용 앱 키
KIWOOM_APP_SECRET=실전시크릿     # 실전용 시크릿
KIWOOM_ACCOUNT_NO=실전계좌번호   # 실전 계좌
```

> **주의**: `config.yaml`의 설정값(n_stocks, min_avg_trading_value 등)도
> 실전 투자금에 맞게 조정해야 합니다.
> 투자금별 최적 설정은 `docs/09_investment_sizing.md`를 참조하세요.

## 6. 모의투자 자금과 실전 자금이 다를 때

### 핵심 결론

모의투자 5억으로 한 번만 테스트하면 됩니다.
금액별로 따로 한 달씩 테스트할 필요는 없습니다.

### 이유

모의투자 테스트의 목적은 **수익률 검증이 아닙니다**.
검증하는 것은 다음 3가지입니다:

1. **시스템 안정성**: 스케줄러가 매일 죽지 않고 돌아가는지
2. **주문 체결**: API 주문 → 체결 → DB 저장 플로우가 정상인지
3. **리밸런싱 로직**: 월말 매도/매수 순서, 교체율 제한이 작동하는지

### 수익률은 백테스트로 확인

```bash
# 1,000만원 기준 백테스트
python -m backtest.engine --cash 10000000

# 1억 기준 백테스트
python -m backtest.engine --cash 100000000
```

백테스트는 과거 데이터로 **임의 금액, 임의 기간** 시뮬레이션이 가능하므로,
모의투자로 금액별 테스트를 반복할 필요가 없습니다.

### 모의투자 자금 변경 방법

키움증권 모의투자 계좌의 초기 자금은 **신청 시 설정**하며,
이후 변경하려면 **모의투자를 해지하고 재신청**해야 합니다.

1. 키움증권 HTS → 모의투자 → 해지
2. 다시 모의투자 신청 → 원하는 금액 설정
3. 새 계좌번호 발급 → `.env` 업데이트

### 권장 테스트 전략

```
[모의투자 5억] → 시스템 검증 (1개월)
    ↓
[백테스트] → 금액별 수익률 비교 (시간 무관)
    ├── python -m backtest.engine --cash 1000000     # 100만원
    ├── python -m backtest.engine --cash 10000000    # 1,000만원
    └── python -m backtest.engine --cash 500000000   # 5억
    ↓
[실전 투자] → 원하는 금액으로 시작
    └── config.yaml 해당 금액에 맞게 조정
```

# Reporting Lag 수정 검증 리포트

**작성일**: 2026-04-15
**배경**: 005620 대성합동지주 폐지 사례(2017-08-25)에서 Reporting Lag가 실전 적용되지 않던
버그가 발견됨. `_get_effective_fundamental_date()`는 정의돼 있었으나 `screen()`에서 호출되지 않아,
분기 보고서 공시 직후 급변한 재무 데이터가 F-Score 필터에 그대로 반영되었다.

## 수정 내역

| 파일 | 변경 |
|------|------|
| `config/settings.py` | `QualityConfig.strict_reporting_lag: bool = True` 추가 |
| `strategy/screener.py` | `screen()`에서 `strict_reporting_lag=True`일 때 `_get_effective_fundamental_date(date)`로 재무 데이터 기준일 전환 |
| `strategy/screener.py` | 폴백 경로도 `fund_query_date` 기준으로 전환 |
| `strategy/screener.py` | 팩터 캐시 키에 `strict_reporting_lag` 포함 (모드 전환 시 오염 방지) |

## 005620 회피 메커니즘 검증

### 수정 전 (lag 미적용)

리밸런싱 기준일 `2017-06-30`의 Q1 보고서 반영 데이터를 사용:
- EPS: +111,049 (3월 결산 Q1 보고서 반영, 5월 중순 공시)
- F-Score: **4/5점** → `min_fscore=4` 필터 통과

### 수정 후 (strict_reporting_lag=True)

재무 데이터 기준일이 `2016-12-29`로 이동 (2016년 연간 보고서, 2017년 3월말 공시):
- EPS: **-3,970** (2016 연간 적자)
- F-Score: **1/5점** → 필터 배제

## 의도치 않은 종목 누락 검증

| 체크 항목 | 결과 |
|-----------|------|
| 선정 종목 수 변화 | 각 리밸런싱 모두 20종목 유지 (동일) |
| 유니버스 크기 변화 | 미미 (2016-12-29 유니버스는 712개, 2017-06-30은 689개 — KRX 상장사 수 자연 변동) |

모든 리밸런싱에서 20종목이 정상 선정되었으므로 **lag 적용이 "유효한 종목"까지 배제하는 부작용은 없음**.

## 결론

✅ **Case A (lag 코드 미작동) 확정 + 수정 완료**
- PRD v2.0 "연간 보고서 +90일 래그" 정책을 실제로 적용
- 005620 유형(분기보고서 공시 직후 급변) 사례 완전 회피
- 전체 2017-2024 기간 **폐지 노출 0건**, 어떤 구간에서도 단일 종목 노출 없음

## 추가 검토 사항 (후속 작업)

- **Value 팩터(PBR, PCR)도 동일한 lag 적용 여부**: 현재 PBR은 당일 주가/BPS 계산이며 BPS는 `fund_query_date` 기준으로 전환되므로 자동 반영됨. 추가 작업 불필요.
- **min_fscore=4 유지**: 005620이 이제 적자 데이터로 배제되므로 min_fscore 상향 불필요.
- **실전 백테스트 성과 재측정**: lag 강화로 최신 분기 반영이 지연되어 전체 CAGR이 소폭 감소할 가능성 있음 — 차후 MultiFactorBacktest 재실행 권장.

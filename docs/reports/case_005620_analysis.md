## 진단

1. **Reporting Lag 미적용**: v2.0에서 연간 보고서 +90일, 분기 +45일 래그를 명시했으나
   본 사례는 **2017-03 결산 반기보고서를 6월 30일에 이미 사용**. 실제로는 8월 중순까지
   래그를 둬야 이 종목은 아직 "5월 적자 상태"로 유지되어 필터에 걸렸을 것.

2. **가치 함정(Value Trap)**: PBR 0.26, PER 0.54는 극단적 저평가 지표인데
   Value 팩터(V=70%)가 이를 높게 평가. 해산 임박 기업의 전형.

3. **F-Score 턱걸이**: 5점 만점 중 정확히 4점으로 필터 통과. 5점 상향 시 배제됨.

## 개선 제안

### 즉시 적용 가능
- **`min_fscore`를 5로 상향** 검토 (현재 4). 단, 정상 종목도 배제될 수 있으므로
  Walk-Forward로 유니버스 축소 영향 측정 필요.
- **거래정지 감지 필터**: 최근 5영업일 중 `volume=0` 비율 ≥ 40% 종목 제외.
  005620은 8월부터 거래정지 → 9월 리밸런싱에서는 자동 배제됨 (하지만 이 종목은
  이미 7월에 매수된 상태).

### 구조적 개선
- **재무 지표 급변 플래그**: EPS 부호 반전 + 변동률 > 150%인 경우 "데이터 신뢰도 낮음"
  플래그 → 스코어에서 감점. DART 최근 공시일과 date의 차이를 확인.
- **Reporting Lag 강화**: v2.0 설계된 래그가 DART 공시일 반영으로 정확히 동작하는지
  검증. 005620처럼 분기 결산 뒤 3개월 내 공시는 실사용 금지.

### 운영 수준
- 본 사례는 **1/32회(3.1%)** 빈도로 발생 → F-Score 4점 유지해도 실전에서 허용 범위.
- 단, 해당 리스크를 받아들이려면 `risk_guard.check_delisting_imminent` 매일 체크와
  관리종목 감지가 필수.

## 결론

005620은 F-Score 4점 필터의 유일한 돌파 사례다. 해산 직전 반기보고서 공시로
**단기적으로만** 재무 지표가 개선되어 필터를 통과했지만, Reporting Lag를 엄격히
적용하면 이 사례는 배제 가능하다. 필터 임계값 변경보다 **Lag 운용 검증**이
더 근본적 해결책으로 보인다.

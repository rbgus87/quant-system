### 7-4. PCR 데이터 가용성 (v2.0 Value 팩터 핵심)

- v2.0 Value 팩터 구성: PBR 50% + PCR 30% + DIV 20%. PCR 누락 시 `value.py`가 PBR+DIV로 가중치 재분배(NaN-aware).
- **DART 분기/반기보고서에 현금흐름표가 없어 PCR = 0% 확보** (KOSPI/KOSDAQ 공통).
- 연간보고서에는 현금흐름표 있으나 DART 호출 로직상 분기/반기 우선 적용됨 → 사실상 PCR 상시 NaN.
- 권고:
  1) `data/dart_client.py`에서 PCR 계산 시 연간보고서(사업보고서) 우선 호출하도록 개선 검토.
  2) 연결재무제표 현금흐름표 우선 시도 후 별도재무제표 폴백.
  3) 확보 불가 시 `value_weights.pcr=0.0` + `psr` 도입 공식화 (PRD_v2 §7).
- 본 백테스트의 V70M30 성과는 사실상 PBR+DIV 2팩터 Value + 모멘텀 체계에서 산출된 것으로 해석해야 함.

### 7-5. F-Score 계산 가능성

- 현 구현은 PER/PBR/DIV 3필드 동시 보유 시 5점 만점 F-Score 계산 가능.
- **실측 (docs/reports/kosdaq_data_quality.md §5)**:
  - KOSPI: 44.0% 레코드만 3필드 동시 보유
  - KOSDAQ: 35.7% 레코드만 3필드 동시 보유
- `min_fscore=4` 필터는 PER/PBR/DIV 중 하나라도 NaN인 종목을 조건부 누락시킴. KOSDAQ에서는 35.7%만 유효 F-Score를 가지므로 필터 후 유니버스 축소 폭이 큼.
- 권고: KOSDAQ 확장 시 (a) `min_fscore=3` 완화 또는 (b) DIV NaN을 0으로 간주하는 로직 추가 검토.

### 7-6. 펀더멘털 누락 종목 처리 방안

- 현 로직: PBR/PER/DIV 중 하나라도 NaN인 경우 `composite.py`의 NaN-aware 가중치 재분배가 보완하지만, 3개 모두 NaN이면 해당 종목은 유효 score 산출 불가 → 자동 탈락.
- 권고:
  1) KOSDAQ 확장 시 `ffill=30d` 옵션 도입 — 최근 30일 내 직전 분기 데이터 폴백.
  2) 신규 상장 종목은 상장 후 2분기(6개월) 대기 후 유니버스 편입.
  3) 펀더멘털 4필드 이상 NaN인 종목은 해당 리밸런싱 유니버스에서 선제 제외.

### 7-8. 확장 여부 권고 (최종)

**결론: 현 시점 KOSDAQ 확장 권고하지 않음.**

근거:
1. **성과 악화**: ALL 시나리오가 KOSPI 단독 대비 CAGR/MDD/Sharpe/Sortino 전면 악화.
2. **PCR 팩터 무효화**: 분기/반기보고서 현금흐름표 부재로 v2.0 Value 팩터 30%가 실질 미작동. 이는 KOSPI에서도 동일한 문제이나, KOSDAQ에서 PBR+DIV 2팩터만으로 저평가 식별이 충분한지 검증 필요.
3. **F-Score 커버리지 저조**: KOSDAQ에서 F-Score 계산 가능 비율 35.7% (KOSPI 44%).
4. **유동성 격차**: 선정 종목 거래대금이 KOSPI 대비 약 1/3. 슬리피지 상향 반영 시 성과 추가 하락 가능.

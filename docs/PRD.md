# PRD — 한국 주식 멀티팩터 퀀트 자동매매 시스템
> **버전**: 1.0 | **작성**: 2025-03 | **개발 환경**: Python 3.11 + Claude Code CLI

---

## 1. 프로젝트 개요

### 1-1. 목적
한국 주식시장(KOSPI)을 대상으로 **밸류 × 모멘텀 × 퀄리티** 3가지 팩터를 결합한 멀티팩터 퀀트 전략을 구현하고, 키움 REST API를 통해 월 1회 자동 리밸런싱을 수행하는 개인용 자동매매 시스템을 개발한다.

### 1-2. 핵심 목표
- 2015~2024년 백테스트 기준 **CAGR 15% 이상, MDD -20% 이내, 샤프 비율 1.0 이상** 달성
- 완전 자동화: 신호 계산 → 주문 실행 → 텔레그램 알림까지 무인 운영
- 소액(100만원) 실전 투입 후 검증 → 단계적 비중 확대

### 1-3. 비기능 요구사항
- **생존 편향 방지**: 과거 날짜 기준 종목 리스트 사용
- **선견 편향 방지**: 월말 신호 계산 → 다음 영업일 시가 체결 가정
- **안정성**: 오류 발생 시 텔레그램 즉시 알림, 주문 미실행 방지
- **보안**: API 키 `.env` 관리, 실전/모의 플래그 이중 확인

## 2. 전략 명세 (Strategy Spec)

### 2-1. 유니버스
| 항목 | 값 | 비고 |
|------|-----|------|
| 대상 시장 | KOSPI | KOSDAQ 제외 (유동성 리스크) |
| 시가총액 하위 컷 | 10% 제외 | 유동성 부족 종목 제거 |
| 제외 섹터 | 금융주 전체 | 은행, 증권, 보험, 기타금융 |
| 제외 조건 | 상장 1년 미만 | 가격 이력 부족 |
| 데이터 소스 | pykrx | KRX 공식 데이터 |

### 2-2. 팩터 정의
| 팩터 | 가중치 | 지표 | 방향 |
|------|--------|------|------|
| 밸류 | **40%** | PBR (0.5) + PER (0.3) + 배당수익률 DIV (0.2) | 낮을수록(PBR/PER) / 높을수록(DIV) 우수 |
| 모멘텀 | **40%** | 12M 수익률 (최근 1개월 제외) | 높을수록 우수 |
| 퀄리티 | **20%** | ROE = EPS/BPS×100 (60%) + 부채비율 역수 (40%) | 높을수록 우수 |

> **모멘텀 계산 기준**: (t-2개월 가격 / t-13개월 가격) - 1  
> 최근 1개월을 제외하는 이유: 단기 반전 효과(Short-term Reversal) 제거

### 2-3. 포트폴리오 구성
| 항목 | 값 |
|------|-----|
| 편입 종목 수 | 30개 |
| 비중 방식 | 동일 비중 (Equal Weight) |
| 리밸런싱 주기 | 매월 마지막 영업일 신호 계산 |
| 체결 기준 | **다음 영업일 시가** (선견 편향 방지) |

### 2-4. 거래 비용
| 항목 | 적용값 | 비고 |
|------|--------|------|
| 수수료 | 0.015% | 매수/매도 공통 |
| 증권거래세 | 0.18% | 매도 시만 적용 (2024년 기준) |
| 슬리피지 | 0.10% | 시장충격 가정 |
| 합계(왕복) | 약 0.51% | 매수+매도 합산 |

## 3. 시스템 구성 (System Components)

### 3-1. 모듈 구성
```
① 데이터 파이프라인   pykrx → SQLite 저장
② 팩터 계산          밸류 / 모멘텀 / 퀄리티 → 복합 스코어
③ 백테스트 엔진      월별 리밸런싱 시뮬레이션
④ 자동매매 연동      키움 REST API (api.kiwoom.com)
⑤ 자동화            APScheduler 월말 트리거
⑥ 알림              텔레그램 봇
⑦ 대시보드          Streamlit
```

### 3-2. 개발 단계 (Phase)
| Phase | 내용 | 기간 |
|-------|------|------|
| 1 | 환경 구축 (Python, 가상환경, Claude Code, Git) | 1~2일 |
| 2 | 데이터 파이프라인 (pykrx 수집, SQLite 저장, 검증) | 3~5일 |
| 3 | 팩터 구현 (Value/Momentum/Quality, 단위 테스트) | 3~5일 |
| 4 | 백테스트 (2015~2024 실행, 성과 분석, 파라미터 최적화) | 3~5일 |
| 5 | 키움 REST API 연동 (모의투자 전체 흐름 검증) | 3~5일 |
| 6 | 자동화 (스케줄러, 텔레그램, Streamlit 대시보드) | 2~3일 |
| 7 | 실전 투입 (소액 100만원 → 3개월 관찰 → 확대) | 진행 중 |

## 6. 리스크 및 주의사항

1. **과최적화**: In-Sample(2015~2020) / Out-of-Sample(2021~2024) 분리 검증 필수
2. **생존 편향**: pykrx의 과거 날짜 기준 종목 조회로 방지 (상장폐지 종목 자동 포함)
3. **데이터 품질**: PBR/EPS 0값, 결측치 → 해당 종목 팩터에서만 제외 처리
4. **API 안정성**: 키움 REST API 장애 시 텔레그램 알림 → 수동 개입
5. **법적 리스크**: 알고리즘 트레이딩은 한국거래소 알고리즘 계좌 자동 등록 대상

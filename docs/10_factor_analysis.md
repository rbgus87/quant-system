# 멀티팩터 전략 구성 분석

> 작성일: 2026-03-13
> 대상: KOSPI 멀티팩터 퀀트 자동매매 시스템

## 1. 팩터 구조 개요

본 시스템은 **밸류(Value) x 모멘텀(Momentum) x 퀄리티(Quality)** 3팩터를 조합한
멀티팩터 전략을 기반으로 합니다.

### 팩터 가중치

| 팩터 | 비중 | 핵심 역할 |
|------|------|----------|
| **밸류 (Value)** | **40%** | 저평가 종목 발굴 |
| **모멘텀 (Momentum)** | **40%** | 추세 추종 |
| **퀄리티 (Quality)** | **20%** | 기업 체질 검증 |

설정 위치: `config/config.yaml` → `factor_weights`

---

## 2. 밸류 팩터 (Value Factor) — 40%

**소스 코드**: `factors/value.py`

저평가된 종목을 발굴하는 팩터입니다. PBR, PER, 배당수익률 3가지 지표를 조합합니다.

### 세부 지표

| 지표 | 팩터 내 비중 | 계산 방식 | 로직 |
|------|-------------|----------|------|
| **PBR** (주가순자산비율) | 50% | 주가 / BPS | 역수 변환 — 낮을수록 고점수 |
| **PER** (주가수익비율) | 30% | 주가 / EPS | 역수 변환 — 낮을수록 고점수, 적자 기업 제외 |
| **배당수익률** (DIV) | 20% | 배당금 / 주가 | 그대로 사용 — 높을수록 고점수 |

설정 위치: `config/config.yaml` → `value_weights`

### 계산 프로세스

```
1. 이상치 처리
   - PBR/PER: 0 이하 제외, 상위 99% 클리핑
   - DIV: 음수 제외

2. 지표별 스코어 생성
   - PBR/PER: 역수(1/x) 변환 → 순위(백분위) → 0~100 스케일
   - DIV: 그대로 → 순위(백분위) → 0~100 스케일

3. 가중 합산 (NaN-aware)
   - 일부 지표 결측 시 가용 가중치만 재분배
   - 예: PER 없는 종목 → PBR 71%, DIV 29%로 재분배

4. 최종 value_score: 0~100 범위
```

---

## 3. 모멘텀 팩터 (Momentum Factor) — 40%

**소스 코드**: `factors/momentum.py`

가격 추세를 추종하는 팩터입니다. 멀티기간 수익률과 듀얼 모멘텀(절대+상대)을 결합합니다.

### 세부 지표 (멀티기간)

| 기간 | 계산 | 비중 | 비고 |
|------|------|------|------|
| **12개월** | t-1개월 종가 / t-12개월 종가 - 1 | **60%** | 최근 1개월 제외 (단기 반전 방지) |
| **6개월** | t-1개월 종가 / t-6개월 종가 - 1 | **30%** | 중기 추세 |
| **3개월** | t-1개월 종가 / t-3개월 종가 - 1 | **10%** | 단기 추세 |

### 듀얼 모멘텀 필터 (절대 모멘텀)

```
조건: 12개월 수익률 ≥ 무위험 수익률 (3.5%)
효과: 하락장에서 모멘텀이 약한 종목 자동 제외 → 현금 대피 효과
```

설정 위치: `config/config.yaml` → `momentum`

### 계산 프로세스

```
1. 수익률 계산
   - 각 기간별 수익률 산출 (최근 1개월은 스킵)

2. 이상치 처리
   - 각 기간별 상하위 1% Winsorize (극단값 클리핑)

3. 스코어 생성
   - 순위(백분위) → 0~100 스케일

4. 가중 합산
   - 12M 60% + 6M 30% + 3M 10%
   - NaN-aware 재분배

5. 절대 모멘텀 필터
   - 12M 수익률 < 3.5% → 해당 종목 제외
```

## 4. 퀄리티 팩터 (Quality Factor) — 20%

**소스 코드**: `factors/quality.py`

기업의 재무 건전성과 수익성을 평가하는 팩터입니다.

### 세부 지표

| 지표 | 비중 | 계산 방식 | 의미 |
|------|------|----------|------|
| **ROE** (자기자본이익률) | 40% | EPS / BPS x 100 | 수익성 |
| **이익수익률** (1/PER) | 30% | 1 / PER | 이익 안정성 |
| **배당 지급 여부** (DIV > 0) | 30% | 배당수익률 값 | 기업 질 신호 |

### F-Score 필터 (5점 만점)

F-Score는 기업의 최소 질적 기준을 확인하는 사전 필터입니다.

| 항목 | 조건 | 배점 |
|------|------|------|
| ROE 양수 | ROE > 0 | +1점 |
| PER 양수 | PER > 0 (흑자) | +1점 |
| 배당 지급 | DIV > 0 | +1점 |
| 저PBR | PBR < 중앙값 | +1점 |
| 고ROE | ROE > 중앙값 | +1점 |

**통과 기준**: 최소 2점 이상 (최악의 기업만 제거)

설정 위치: `config/config.yaml` → `quality`

### 계산 프로세스

```
1. ROE 스코어
   - BPS > 0인 종목만 (자본잠식 제외)
   - ROE 범위 클리핑: -50% ~ +100%
   - 순위 → 0~100 스케일

2. 이익수익률 스코어
   - PER > 0인 종목만 (적자 기업 제외)
   - 1/PER 계산 후 순위 스코어

3. 배당 스코어
   - DIV 값을 그대로 사용 (미지급도 0으로 포함)
   - 순위 스코어

4. F-Score 사전 필터
   - 2점 미만 종목 제외
```

## 5. 복합 팩터 스코어 (Composite Score)

**소스 코드**: `factors/composite.py`

3개 팩터를 가중 합산하여 최종 종합 점수를 산출합니다.

### 종합 점수 공식

```
Composite Score = Value(40%) + Momentum(40%) + Quality(20%)
```

### NaN-aware 가중치 재분배

| 가용 팩터 | 가중치 적용 |
|----------|-----------|
| 3개 모두 | V 40%, M 40%, Q 20% (원래대로) |
| 2개만 | 있는 팩터의 가중치를 합 1.0으로 재분배 |
| 1개만 | 해당 팩터 점수 = 최종 점수 |

### 최소 팩터 수 요구

- 펀더멘털 데이터 있을 때: 2개 이상
- 펀더멘털 없을 때 (모멘텀 전용 폴백): 1개

## 6. 유니버스 필터링

**소스 코드**: `data/processor.py`

팩터 계산 전에 투자 대상 종목을 걸러내는 필터입니다.

### 필터링 순서

| 순번 | 필터 | 기준 | 설정값 |
|------|------|------|--------|
| 1 | 거래정지 | 거래정지 여부 | 제외 |
| 2 | 시가총액 | 하위 N% | 10% 이상만 (하위 10% 제외) |
| 3 | 금융주 | WICS 섹터 | 은행, 증권, 보험, 기타금융, 다각화금융 제외 |
| 4 | 상장일수 | 최소 N일 | 365일 이상 |
| 5 | 유동성 | 20일 평균 거래대금 | 2억원 이상 |

설정 위치: `config/config.yaml` → `universe`

### 추가 필터 (스크리너 내)

| 필터 | 기준 | 설정값 | 설정 위치 |
|------|------|--------|----------|
| 변동성 필터 | 연환산 변동성 | 상위 20% 제외 | `volatility` |
| F-Score 필터 | 기업 질 지표 | 2점 이상 | `quality` |
| 절대 모멘텀 | 12M 수익률 | 3.5% 이상 | `momentum` |

## 7. 스크리닝 파이프라인

**소스 코드**: `strategy/screener.py`

전체 종목 선정 과정을 순서대로 정리합니다.

```
MultiFactorScreener.screen(date, market, n_stocks)

  [1] 데이터 수집
      ├── get_fundamentals_all() → PER, PBR, EPS, BPS, DIV
      ├── get_market_cap()       → 시가총액
      └── get_avg_trading_value() → 평균 거래대금

  [2] 유니버스 필터 (processor.filter_universe)
      ├── 시가총액 하위 10% 제외
      ├── 금융주 제외
      └── 유동성 필터 (2억원 이상)

  [3] 변동성 필터
      └── 고변동성 상위 20% 제외

  [4] 팩터 계산
      ├── ValueFactor.calculate()    → value_score (0~100)
      ├── QualityFactor.calculate()  → quality_score (0~100)
      │   └── F-Score 필터 (2점 이상만 통과)
      └── MomentumFactor.calculate() → momentum_score (0~100)
          └── 절대 모멘텀 필터 (3.5% 이상)

  [5] 복합 스코어 (MultiFactorComposite.calculate)
      └── composite_score = V(40%) + M(40%) + Q(20%)

  [6] 최종 포트폴리오 선정
      ├── 상위 N개 종목 선정 (기본 10개)
      └── 동일 가중치 (1/N)
```

## 8. 리밸런싱 및 위험 관리

**소스 코드**: `backtest/engine.py`

### 리밸런싱 주기

```
리밸런싱일: T = 월 마지막 영업일
  1. T일: 팩터 계산 및 포트폴리오 결정
  2. T+1 영업일: 시가(open)로 매매 체결 (선견 편향 방지)
```

### 위험 관리 계층 (4단계)

| 계층 | 메커니즘 | 기준 | 설정값 | 현재 상태 |
|------|---------|------|--------|----------|
| 1 | 트레일링 스톱 | 매수가 대비 하락폭 | -20% | 활성 |
| 2 | 시장 레짐 필터 | KOSPI 200일 이평선 | 중립 70%, 약세 50% | 활성 |
| 3 | 변동성 타겟팅 | 실현 변동성 | 연환산 12% | 비활성 (0.99) |
| 4 | MDD 서킷브레이커 | 고점 대비 하락폭 | -30% | 비활성 (0.99) |

설정 위치: `config/config.yaml` → `trading`, `market_regime`

> **참고**: 변동성 타겟팅과 MDD 서킷브레이커는 현재 0.99로 사실상 비활성화.
> 밸류 전략 특성상 하락장에서 리밸런싱이 유리하기 때문.

## 9. 데이터 수집 흐름

**소스 코드**: `data/collector.py`

### 조회 우선순위

```
기본 지표 (PER, PBR, EPS, BPS, DIV):
  1순위: SQLite 캐시 (이전 수집분)
  2순위: KRX Open API (KRX_OPENAPI_KEY)
  3순위: DART OpenAPI (DART_API_KEY)
  4순위: pykrx 폴백

OHLCV (시가/고가/저가/종가/거래량):
  1순위: SQLite 캐시
  2순위: pykrx (개별 종목만 가능)

시장 전체 데이터:
  KRX Open API: 전체 종목 벌크 조회
```

### 데이터 정제 규칙

| 지표 | 정제 규칙 |
|------|----------|
| PBR, PER | 0 이하 제외, 상하위 1% Winsorize |
| DIV (배당수익률) | 음수 제외 |
| BPS (주당순자산) | 0 이하 (자본잠식) → NaN 처리 |
